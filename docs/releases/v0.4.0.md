# Mnemos 0.4.0: Trust Release for Claude Code

Mnemos 0.4.0 makes the default Claude Code path simpler to trust. This release hardens startup and config recovery on the shipped SQLite setup, turns recall plus curator into a clearer day-to-day workflow, improves inspectability so users can see why memories were stored and retrieved, and backs the SQLite path with repeatable claim-driving benchmark runs. The product story is now one file, one install path, one scoped memory workflow.

## Highlights

- actionable MCP startup errors instead of dead handshakes on stale or unsupported config
- shipped `mnemos-recall` and `mnemos-curator` workflows for Claude Code
- richer retrieval inspection with query-aware relevance explanations
- repeat-run SQLite benchmark reporting with stable production-gate enforcement
- tighter docs around SQLite-first local memory and honest host support boundaries

## Benchmark summary

- repeated `claim-driving` gate: `2/2` runs passed
- SQLite `mrr_lift_ratio` range: `0.40` to `1.67`
- SQLite `latency_p95_ratio` range: `1.05` to `2.49`

## Why this release matters

Claude Code already has built-in memory. Mnemos is for users who want stronger repo and workspace scope boundaries, inspectable retrieval behavior, and a local SQLite-backed continuity layer they can understand instead of treat as a black box.
